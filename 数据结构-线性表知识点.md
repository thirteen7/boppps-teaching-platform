# 数据结构：线性表知识点

## 1. 基本概念

线性表是由零个或多个数据元素组成的有限序列。除第一个元素外，每个元素有且仅有一个直接前驱；除最后一个元素外，每个元素有且仅有一个直接后继。

线性表的特点：

- 元素之间存在一对一的线性关系。
- 元素排列有先后顺序。
- 表中元素个数称为线性表的长度。
- 长度为 0 的线性表称为空表。

## 2. 线性表的基本操作

| 操作 | 含义 |
| --- | --- |
| 初始化 | 创建一个空线性表 |
| 求表长 | 获取线性表中元素个数 |
| 查找元素 | 根据值或位置查找元素 |
| 插入元素 | 在指定位置插入新元素 |
| 删除元素 | 删除指定位置或指定值的元素 |
| 遍历线性表 | 依次访问表中所有元素 |
| 判断空表 | 判断线性表是否为空 |

## 3. 顺序表

顺序表是线性表的顺序存储结构，通常使用一段连续的存储空间保存元素。

### 优点

- 支持随机访问。
- 根据下标访问元素的时间复杂度是 O(1)。
- 存储密度高，不需要额外指针空间。

### 缺点

- 插入和删除可能需要移动大量元素。
- 容量通常需要提前分配。
- 扩容可能带来额外开销。

### 常见时间复杂度

| 操作 | 时间复杂度 |
| --- | --- |
| 按下标访问 | O(1) |
| 查找指定值 | O(n) |
| 插入元素 | O(n) |
| 删除元素 | O(n) |

## 4. 链表

链表是线性表的链式存储结构，元素通过节点存储，节点之间通过指针或引用连接。

一个节点通常包含：

- 数据域：保存元素数据。
- 指针域：保存下一个节点的位置。

### 单链表

单链表中每个节点只保存后继节点的地址。

优点：

- 插入和删除时不需要整体移动元素。
- 不要求连续存储空间。

缺点：

- 不能直接随机访问。
- 查找某个位置的元素需要从头节点开始遍历。

### 双链表

双链表中每个节点既保存前驱节点地址，也保存后继节点地址。

优点：

- 可以双向遍历。
- 删除指定节点时更方便。

缺点：

- 每个节点需要更多指针空间。
- 实现比单链表更复杂。

### 循环链表

循环链表的最后一个节点指向第一个节点，适合需要循环处理数据的场景。

## 5. 顺序表与链表对比

| 对比项 | 顺序表 | 链表 |
| --- | --- | --- |
| 存储空间 | 连续 | 不一定连续 |
| 随机访问 | 支持，O(1) | 不支持，通常 O(n) |
| 插入删除 | 可能需要移动元素 | 修改指针即可 |
| 空间开销 | 较小 | 需要额外指针 |
| 容量限制 | 可能需要扩容 | 更灵活 |
| 适用场景 | 查询多、修改少 | 插入删除多 |

## 6. 线性表的应用

- 学生成绩表。
- 通讯录。
- 图书目录。
- 顺序存储的数组结构。
- 链式存储的任务列表。
- 栈和队列等受限线性表。

## 7. 易错点

- 线性表强调逻辑结构，不等于数组。
- 顺序表和链表是线性表的两种常见存储方式。
- 顺序表随机访问快，但插入删除可能慢。
- 链表插入删除灵活，但查找和随机访问效率低。
- 第一个元素没有直接前驱，最后一个元素没有直接后继。
- 空表也是合法的线性表。

## 8. 简单示例

```text
线性表 L = (A, B, C, D)

A 是第一个元素，没有直接前驱。
B 的直接前驱是 A，直接后继是 C。
C 的直接前驱是 B，直接后继是 D。
D 是最后一个元素，没有直接后继。
```

如果在位置 3 插入元素 X：

```text
插入前：A B C D
插入后：A B X C D
```

如果删除元素 B：

```text
删除前：A B X C D
删除后：A X C D
```
