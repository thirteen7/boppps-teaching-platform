# 数据结构：队列知识点

## 1. 基本概念

队列是一种受限的线性数据结构，只允许在一端插入元素，在另一端删除元素。

- 插入元素的一端称为队尾。
- 删除元素的一端称为队头。
- 队列遵循先进先出原则，即 FIFO。

生活中的排队买票、打印任务排队、消息队列等场景都体现了队列思想。

## 2. 队列的基本操作

| 操作 | 含义 |
| --- | --- |
| 初始化队列 | 创建一个空队列 |
| 入队 | 在队尾插入新元素 |
| 出队 | 删除并返回队头元素 |
| 取队头元素 | 查看队头元素但不删除 |
| 判断队空 | 判断队列中是否没有元素 |
| 判断队满 | 顺序存储时判断队列是否已满 |

## 3. 队列的存储结构

### 顺序队列

顺序队列使用数组存储元素，通常设置两个指针：

- `front`：指向队头位置。
- `rear`：指向队尾后一个位置。

普通顺序队列的问题是：随着入队和出队不断进行，数组前面的空间可能被浪费，出现“假溢出”。

### 循环队列

循环队列将数组首尾相接，把数组空间看作一个环。

常用判断方式：

- 队空：`front == rear`
- 队满：`(rear + 1) % maxSize == front`

为了区分队空和队满，循环队列通常会浪费一个存储单元。

### 链式队列

链式队列使用链表实现，通常包含：

- 队头指针 `front`
- 队尾指针 `rear`

链式队列不需要预先固定容量，适合元素数量变化较大的场景。

## 4. 时间复杂度

| 操作 | 时间复杂度 |
| --- | --- |
| 入队 | O(1) |
| 出队 | O(1) |
| 取队头元素 | O(1) |
| 查找某个元素 | O(n) |

队列的优势是队头删除和队尾插入效率高，但不适合频繁随机访问。

## 5. 队列的应用场景

- 操作系统中的进程调度。
- 打印机任务管理。
- 网络请求排队。
- 广度优先搜索。
- 消息队列和任务队列。
- 缓冲区管理。

## 6. 与栈的区别

| 结构 | 插入位置 | 删除位置 | 原则 |
| --- | --- | --- | --- |
| 栈 | 栈顶 | 栈顶 | 后进先出 LIFO |
| 队列 | 队尾 | 队头 | 先进先出 FIFO |

栈像一摞盘子，后放上去的先取走；队列像排队，先来的人先办理。

## 7. 易错点

- 队列不是随机访问结构，不能像数组一样高效访问任意位置。
- 顺序队列可能出现假溢出，循环队列可以解决这个问题。
- 循环队列中 `rear` 不一定比 `front` 大，因为下标会循环。
- 队满和队空条件不能混淆。
- 链式队列一般不会出现固定容量导致的队满问题，但会受到内存限制。

## 8. 简单示例

```text
初始队列：空

入队 A：A
入队 B：A B
入队 C：A B C

出队：A
剩余队列：B C

再入队 D：B C D
```

可以看到，最先进入队列的 `A` 最先离开。
